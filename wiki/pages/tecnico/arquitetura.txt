===== Visão de alto nível =====

3 containers orquestrados via Docker Compose em uma única VM no :

  * **''bagre-db''** — PostgreSQL 16, dados em volume nomeado ''bagre_db_data''
  * **''bagre-api''** — Fastify + Prisma, expõe ''/api/*'' e ''/metrics''. Acessível **somente** pela rede interna do Docker (porta ''3001'' não está mapeada no host)
  * **''bagre-web''** — nginx serve o build estático do React **e** faz proxy ''/api/* → api:3001''. Único container acessível de fora (portas 80 e 443)

===== Fluxo de uma requisição =====

<code>
Browser (https://10.0.0.10)
  └─► nginx [443 TLS]                                    (container web)
        ├─► serve /usr/share/nginx/html/* (SPA)
        └─► proxy_pass http://api:3001/api/*             (container api)
              └─► Fastify (JWT verify)
                    └─► Prisma → PostgreSQL [5432]       (container db)
</code>

===== Proteção global (auth) =====

''apps/api/src/index.js'' instala um hook ''onRequest'' que:

  - Pula URLs em ''PUBLIC'' (whitelist: ''/api/health'', ''/api/config'', ''/api/auth/login'', ''/api/auth/signup'', ''/api/auth/reset-request'', ''/api/auth/reset'', ''/api/auth/sso/start'', ''/api/auth/sso/callback'', ''/api/import/seed'', ''/api/ingest/*'', ''/metrics'')
  - Faz ''jwtVerify()'' no token Bearer
  - Carrega ''User'' do banco (verifica ''active'')
  - Bloqueia métodos de escrita (''POST/PATCH/PUT/DELETE'') para perfis ''!= ADMIN'', **exceto** ''/api/auth/change-password'' e ''/api/auth/me''

→ READER é read-only **por design global**, sem precisar pingar isso em cada rota.

===== Endpoints especiais =====

  * **''/api/ingest/*''** — usam token próprio (header ''X-Ingest-Token'' = env ''INGEST_TOKEN''). Para tooling externo (Zabbix push, scanners). Não usam JWT.
  * **''/api/stats''** — env-gated: protegido por default; vira público se ''STATS_PUBLIC=true'' (debug only).
